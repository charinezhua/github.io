Hexo命令--之二，备查。 以下命令需要在本地Blog路径下执行

Step1 创建文章

layout 可选参数，用以指定文章类型，若无指定则默认由_config.yml文件的default_layout决定 title 必填参数，用以指定文章标题，如果title中含有空格需要使用双引号包围

Step2 编辑文章

由于在首页中显示文章内容使用的是 post.content 我们如果想在主页中实现文章的部分内容渲染，需要在文章中添加一个 <!-- more -->标记。添加了这个标记之后，post.excerpt将会获取到标记之前的内容。之后的内容大家可以随意DIY啦！ 编辑md文件建议使用Typora软件 特别推荐！！！

关于文章添加图片：

Step3 清除缓存 （可省略 建议使用

清除缓存文件 db.json 和已生成的静态文件 public 当网站显示异常时可以执行这条命令试试

Step4 生成静态文件

选项 描述
-d 生成后部署，与 hexo d -g等价

这步骤就相当于程序的编译，如果有错误的话，在终端会有相应的提示信息 如果没有错误的话 可以继续往下执行

Step5 启动本地服务器 （可省略）

选项 描述
-p 指定服务器端口，默认为 4000
-i 指定服务器IP地址，默认为 0.0.0.0
-s 静态模式 ，仅提供public文件夹中的文件并禁用文件监视

用于预览主题，默认地址： http://localhost:4000/

Step6 部署到远程站点

选项 描述
-g 生成后部署，与 hexo g -d等价

说明 ：部署前需要修改 _config.yml 配置文件，下面以 git 为例说明

其他相关命令

hexo --safe ——安全模式

禁用加载插件和脚本，安装新插件时遇到问题可尝试此操作

hexo --debug ——调试模式

将详细消息记录到终端和debug.log 文件

hexo --silent ——静默模式

静默输出到终端

详细信息请参考：https://hexo.io/docs/deployment.html
